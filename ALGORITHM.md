## 1. 背景与动机

### 原始系统 (v1.0)

第一版基于简单的 **Hot/Cold 分类**：
- 统计历史出现频率
- 分为 Hot (频繁)、Cold (稀少) 两类
- 按策略比例 A/B/C/D 随机生成号码

**测试结果 (8张票):**
```
T02 [B-cold] → 0+1  ⭐ WON $2
T07 [D-hybrid] → 1+1  ⭐ WON $4
```
胜率：**25%** (2/8)，总奖金：$6

### 发现的问题

> 全局频率忽略了数字在**不同排序位置**上的分布规律。

例如，数字 `36` 历史上频繁出现，但若它总是落在**第3位(中间位置)**，而你把它排在第1位 → 该组合的历史吻合度很低。

**位置向量**捕捉这个维度：`P(number | sorted_position)` 而非 `P(number)`。

## 2. 算法架构总览

```
历史数据 (N期开奖记录)
         │
         ├──► Layer 1: 全局频率表 freq[num] → Hot/Cold 分类
         │
         ├──► Layer 2: 位置频率表 pos_table[pos][num] ← 核心新增
         │
         └──► Layer 3: MB频率表 mb_freq[mb_num]
                              │
                              ▼
                   候选池生成 (n_candidates = 500)
                   按位置概率加权采样
                              │
                              ▼
                   综合评分 Score = α·HC + β·Pos + γ·MB
                              │
                              ▼
                   Top-N 过滤 → 策略分组 → 输出票面
```

## 3. Layer 1 — Hot/Cold 频率分析

### 公式

对于白球号码 `n`，其加权频率为：

$$\text{freq}(n) = \frac{\sum_{i=1}^{N} \delta(n \in \text{draw}_i) \cdot \lambda^{N-i}}{\sum_{\text{all}} \text{counts}}$$

其中 `λ = decay = 0.97`（最近期权重更高）

### 分类规则

| 分类 | 定义 |
|------|------|
| **Hot** | 全局频率排名前15名的号码 |
| **Cold** | 全局频率排名后15名的号码 |
| **Neutral** | 中间段号码 |

### 评分贡献

$$S_{HC} = \frac{1}{5}\sum_{j=1}^{5} \text{freq}(n_j)$$

## 4. Layer 2 — 位置向量概率 (核心新增)

### 核心思想

将每次历史开奖的5个白球**从小到大排序**后，记录每个号码出现在哪个**序位**：

```
历史开奖: [8, 36, 23, 68, 52]
排序后:   [8, 23, 36, 52, 68]
               ↓
  P1←8, P2←23, P3←36, P4←52, P5←68
```

对每个位置独立建立概率分布：

$$P(\text{num} \mid \text{pos}=k) = \frac{\sum_{i} \delta(\text{sorted\_draw}_i[k] = \text{num}) \cdot \lambda^{N-i}}{\sum_{\text{all}} \text{weight at pos } k}$$

### 位置分布示例（来自实际运行）

| 位置 | Top-5 高概率号码 |
|------|----------------|
| P1 (最小) | 8(11.3%), 5(10.8%), 10(10.4%), 2(10.3%), 7(9.9%) |
| P2 | 23(11.3%), 19(10.4%), 25(10.4%), 22(9.7%), 18(9.4%) |
| P3 (中间) | 36(11.3%), 33(10.7%), 37(10.4%), 35(9.7%), 31(8.2%) |
| P4 | 47(11.4%), 52(10.7%), 44(10.6%), 51(10.1%), 46(9.9%) |
| P5 (最大) | 68(15.7%), 60(14.0%), 63(11.1%), 65(10.4%), 66(10.1%) |

> P5 的集中度明显高于其他位置 → 大号码具有位置粘性

### 评分：几何平均位置概率

使用**几何平均**而非算术平均，是因为几何平均会**惩罚任何单一位置的极低概率**：

$$S_{Pos} = \exp\left(\frac{1}{5}\sum_{k=1}^{5} \ln P(n_k \mid \text{pos}=k)\right)$$

归一化到 [0, 1]：

$$S_{Pos\_norm} = \frac{S_{Pos}}{S_{Pos}^{\text{best}}}$$

其中 `best` 是每个位置选择最高频号码所得的上界。

### 为什么用几何平均而非算术平均？

```
例子 A: 位置概率 = [0.10, 0.10, 0.10, 0.10, 0.10]
  算术均值 = 0.10
  几何均值 = 0.10

例子 B: 位置概率 = [0.001, 0.15, 0.15, 0.15, 0.15]  ← P1极低
  算术均值 = 0.122  (看起来还行)
  几何均值 = 0.016  (正确反映了P1是异常值)
```

几何平均强迫每个位置都合理，单个"奇怪"位置会拖低整体评分。

## 5. Layer 3 — Mega Ball 频率

$$S_{MB} = \frac{\text{mb\_freq}(mb)}{\max(\text{mb\_freq})}$$

Mega Ball 独立于白球，使用与 Layer 1 相同的指数衰减加权频率，然后归一化。

---

## 6. 综合评分公式

$$\boxed{Score = \alpha \cdot S_{HC} + \beta \cdot S_{Pos} + \gamma \cdot S_{MB}}$$

### 默认权重

| 参数 | 值 | 含义 |
|------|-----|------|
| `α` (alpha) | **0.35** | Hot/Cold 层权重 |
| `β` (beta)  | **0.45** | 位置向量层权重 ← 主导层 |
| `γ` (gamma) | **0.20** | Mega Ball 层权重 |

> **为什么 β 最大？** 位置向量是最精细的特征，它同时包含了频率信息（高频号码在其常见位置上概率也高）加上位置信息。α+β+γ = 1.0。

## 7. 候选池生成算法

不用 DFS，不用暴力枚举（C(70,5) ≈ 1,200万种组合太慢），采用**位置加权采样 + 候选池过滤**：

### 算法步骤

```python
# 伪代码
for _ in range(500):                     # 步骤1: 生成500个候选
    combo = []
    used = set()
    for pos in range(5):                 # 步骤2: 按位置顺序采样
        candidates = band[pos] - used    # 软区间限制重叠
        weights = [pos_table[pos][n]     # 步骤3: 位置概率加权
                   for n in candidates]
        num = weighted_random(candidates, weights)
        combo.append(num)
        used.add(num)
    mb = weighted_random(mb_candidates, mb_weights)
    ticket = score(combo, mb)            # 步骤4: 三层评分
    pool.append(ticket)

pool.sort(by=score, descending=True)     # 步骤5: 排序取Top-N
return pool[:8]
```

### 软区间 (Soft Bands)

| 位置 | 数字范围 | 说明 |
|------|---------|------|
| P1 | 1–15 | 最小号段（大幅重叠允许） |
| P2 | 10–30 | 中低号段 |
| P3 | 20–50 | 中间号段 |
| P4 | 35–60 | 中高号段 |
| P5 | 45–70 | 最大号段 |

各区间设计有重叠，保证自然号段分布，避免强制排序约束导致采样偏差。

### 复杂度分析

| 步骤 | 时间复杂度 |
|------|-----------|
| 建位置表 | O(N × 5) |
| 建频率表 | O(N × 5) |
| 单候选生成 | O(5 × B) ≈ O(1)，B = band大小 |
| 500候选评分 | O(500 × 5) |
| 排序 | O(500 log 500) |
| **总计** | **O(N + 500·5)** ← 极快 |

## 8. 策略分类 A/B/C/D

| 代码 | 名称 | 条件 | 特点 |
|------|------|------|------|
| A-hot | 热号优先 | Hot号 ≥ 3个 | 追高频，回撤风险 |
| B-cold | 冷号逆投 | Cold号 ≥ 3个 | 反转策略，高回报/低概率 |
| C-random | 均衡随机 | 不满足以上 | 位置导向为主 |
| D-hybrid | 冷热混合 | Hot ≥ 2 且 Cold ≥ 1 | 平衡策略 |

### 默认策略分组比例

### 调优建议

## 12. 局限性声明

> [!IMPORTANT]
> **彩票是真随机过程（i.i.d.）。** 任何历史统计方法均无法从数学上证明能改变概率。

本算法的实际价值在于：

1. ✅ **过滤极低概率组合** — 排除历史上从未出现、统计上极"异常"的号码组合
2. ✅ **结构化选号** — 避免全集中在某一数字段（如5个号码都在1-20）
3. ✅ **可解释性** — 每张票都有量化的评分，便于复盘和优化
4. ❌ **无法预测具体号码** — 位置概率基于历史，但每期开奖独立
5. ❌ **期望值为负** — 彩票票面期望永远 < 票价（这是数学事实）

**最诚实的说法：** 这个算法让我们更系统、更有依据地玩彩票，而不是随机乱选。
